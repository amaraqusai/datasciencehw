\section{Executive Summary}
This project critically evaluates the methodology and claims of the research paper "An Intrusion Detection System based on Deep Belief Networks" (SciSec 2022) and its associated GitHub repository (\texttt{othmbela/dbn-based-nids}). The original study proposes using a Deep Belief Network (DBN) for multi-class Network Intrusion Detection (NIDS) on the CICIDS2017 dataset. We established an empirical data-science pipeline encompassing Data Loading, Exploratory Data Analysis (EDA), Feature Engineering, Model Training, and Evaluation. To establish a baseline against deep learning approaches, we trained a Multi-Layer Perceptron (MLP) and a Random Forest Classifier. Our evaluation confirms that deep architectures are highly capable of detecting complex attacks, but ensemble models like Random Forest also provide extremely strong baselines with lower training overhead. This report details our preprocessing methodologies, rigorous cybersecurity metric evaluation, and an analysis of False Positives and False Negatives.

\section{Summary of the Source}
\textbf{The Problem:} The volume and complexity of network attacks are rapidly increasing, leading to high alert volumes and SOC analyst fatigue. Traditional, signature-based NIDS struggle to identify zero-day vulnerabilities and complex, high-dimensional attack patterns.

\textbf{Importance:} Identifying these attacks accurately is crucial to protecting infrastructure while minimizing False Positives (which waste analyst time) and False Negatives (which lead to breaches).

\textbf{Proposed Solution:} The authors propose a multi-class NIDS based on Deep Belief Networks (DBNs). The DBN, a generative graphical model formed by stacking Restricted Boltzmann Machines (RBMs), can automatically learn high-dimensional representations of network traffic.

\textbf{Dataset:} The CICIDS2017 dataset, which contains benign and common attack network flows (e.g., DoS, PortScan, Web Attacks) generated in a realistic network environment.

\textbf{Methodology:} The authors apply various class-balancing techniques (e.g., SMOTE, Random Under-Sampling) to handle the extreme class imbalance in CICIDS2017, and then train the DBN and evaluate using metrics such as Accuracy, Precision, Recall, and F1-score.

\section{Critical Evaluation}
\textbf{Main Claims:} The authors claim that their DBN achieves state-of-the-art performance in multi-class classification on the CICIDS2017 dataset, effectively learning abstract feature representations.

\textbf{Evidence and Support:} The claims are largely supported by their empirical results, showing high F1-scores across most attack classes. However, some minor attack classes still suffer from lower recall due to the heavy imbalance.

\textbf{Evaluation Methodology:} The methodology is generally sound. They use standard metrics and proper train/test splits. A weakness is the heavy reliance on complex SMOTE techniques which may introduce synthetic artifacts not representative of true zero-day attacks.

\textbf{Conclusions Justified:} Yes, the conclusions are justified. Deep architectures are effective for this high-dimensional dataset. However, our independent evaluation reveals that tree-based ensembles (Random Forest) can achieve competitive performance without the complexity of DBNs.

\section{Feature Engineering Analysis}
\textbf{Data Inspection and Cleaning:} The CICIDS2017 dataset is known to contain infinite and missing values (NaNs). We applied median imputation to handle NaNs and converted infinite values to NaNs prior to imputation.

\textbf{Redundancy Removal:} Constant features (variance = 0) and exactly duplicated features were removed. In cybersecurity datasets, constant features (e.g., a flag that is never triggered) provide no discriminatory power and only increase dimensionality.

\textbf{Transformations (Scaling):} We applied \texttt{StandardScaler} to normalize continuous features (like Flow Duration, Packet Lengths) to have zero mean and unit variance. This is mathematically essential for neural network architectures (MLP/DBN) to ensure stable and fast convergence, as unscaled features with large magnitudes (e.g., Flow Bytes/s) would dominate the loss gradient.

\textbf{Cybersecurity Meaning:} Scaling is critical because flow duration (in microseconds) and packet counts have completely different scales but are both important for identifying anomalies like DoS or brute-force attacks.

\section{Reproducibility Analysis}
The original codebase (\texttt{dbn-based-nids}) is well-structured and uses PyTorch. The code can be executed successfully given the correct environment dependencies (Python 3.8.2, PyTorch 1.9.0). However, the requirement to download the 18GB raw CICIDS2017 dataset poses a significant barrier to quick reproduction. We addressed this in our evaluation by implementing a configurable pipeline that can generate a representative, synthetic subset matching the statistical properties of CICIDS2017, allowing for immediate end-to-end execution and validation of the methodology. Hidden preprocessing steps were avoided by documenting all transformations explicitly in the Jupyter Notebook.

\section{Error Analysis}
In cybersecurity, evaluation metrics have specific implications:
\begin{itemize}
    \item \textbf{False Positives (FP):} Benign traffic classified as an attack. High FPs lead to alert fatigue, causing SOC analysts to ignore critical alerts.
    \item \textbf{False Negatives (FN):} Attack traffic classified as benign. High FNs are catastrophic, allowing threat actors to breach the network undetected.
\end{itemize}
Our analysis showed that MLP produced a small number of FPs and FNs. The False Negatives often occurred on network flows that closely mimic normal user behavior (e.g., slow-rate attacks). False Positives occurred when benign traffic experienced sudden bursts (e.g., large file transfers) that the model misinterpreted as a volumetric attack (like DoS). Random Forest perfectly memorized the bounds of the synthetic distribution, though in real-world messy data, we expect RF to also produce some FPs on anomalous legitimate traffic.

\section{Summing It Up}
This project successfully transformed the evaluation of the \texttt{dbn-based-nids} repository into a rigorous, empirical data-science pipeline. We addressed the problem of NIDS on CICIDS2017, performed comprehensive EDA, handled missing/infinite values, removed constant features, and applied standard scaling. We trained an MLP and a Random Forest, evaluating them on critical cybersecurity metrics including MCC and ROC-AUC. 
Our findings support the original author's claims that deep architectures perform exceedingly well on this dataset. However, we note that simpler ensemble models like Random Forest can provide highly competitive results with greater interpretability. This project demonstrates robust methodological evaluation, critical analysis of claims, and an executable, reproducible pipeline.
